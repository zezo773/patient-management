# API Requests

This directory contains HTTP request collections used to manually test and validate backend endpoints.

## Structure

- **`patient-service/`** – endpoint request files specific to the patient service

## How to Use

Open `.http` files in a compatible HTTP client (such as IntelliJ HTTP Client or VS Code extensions) and run requests directly.

## Best Practices

- Keep example payloads realistic and up to date
- Reflect API contract changes immediately
- Use descriptive request names for easier maintenance
