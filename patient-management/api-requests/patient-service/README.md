# Patient Service API Requests

Request examples for validating patient-related REST endpoints.

## Included Requests

- **`create-patient.http`** – create a new patient
- **`get-patients.http`** – retrieve patient list/data
- **`update-patient.http`** – update an existing patient
- **`delete-patient.http`** – remove a patient

## Usage Notes

- Ensure the service is running before sending requests
- Update host/port if your local configuration differs
- Keep IDs and payload fields aligned with your current DTO/domain model
